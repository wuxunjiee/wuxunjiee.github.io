# 真机调试运行

前置操作

  • 在手机设置中找到开发者选项 以华为 8X 为例，设置->关于手机->连击 7 次版本号->输入密码->返回->系统更新->开发人员选项->勾选开发人员选项及 usb 调试（其他型号手机百度对应手机型号打开开发者选项）

  • 我是用华为 8X 时开启 usb 调试后总是自动关闭，将“仅充电“模式下允许 ADB 调试选项勾选。这样就不会出现这个问题了。如无此问题可以忽略这步操作

真机调试

  • ios

    • 下载安装 iTunes
    • 使用数据线连接到电脑
    • 确保 iTunes 能正常连接到手机
    • 它会在手机中安装一个用于调试的测试 App: Hbuilder
    • 在手机上设置 Hbuilder 为受信任的开发者

  • 安卓 与 ios 操作类似不需要下载安装 Itunes

    • 另外 安卓手机可以安装：Vysor 苹果手机可以安装：AirServer 可以把手机投屏到电脑上，方便开发调试

# 打包发布

  • 配置 manifest.json 文件
  • 在 HBuilder 中找到：发行->原生 App(云打包)
  • 等待一段时间后，得到打包结果安装包，然后安装到手机上测试
  • 最后根据需要发布到对应的手机应用商店

打包测试方式

项目中的文件都被打包到应用安装包中了，只要安装了这个应用就不需要请求下载这些文件，除非是一些请求接口的功能必须是联网的。这种方式好处是一些核心文件不需要联网下载，但是更新麻烦，如果你修改了代码，则需要重新打包

还有另外一种方式，我们把应用部署为网站，然后在这个壳儿里面加载我们应用的那个网址

如果是开发测试，则将 manifest.json 文件中的lanuchpath修改为你的局域网地址

如果是发布部署，则将lanuchpath修改为你的线上地址

优缺点

两种方式优缺点：第一种流畅一些，可以少请求一些资源，节约流量。第二种方便开发人员更新 App 内容，将代码推送上去，自动部署，网站就更新了。

Vue搭建的项目怎么打包

如果是Vue CLI搭建的项目 则在项目根目录添加app文件夹，并在里面创建manifest.json即可完成Vue项目的打包发布

产生的问题

手机 App 的网页可能会有缓存问题：

解决办法为前后端禁用缓存，这也是为什么网站更新上线选择在夜深人静的时候的其中一个原因

前端 html 页面添加如下代码

<meta http-equiv="pragram" content="no-cache" />
<meta http-equiv="cache-control" content="no-cache, must-revalidate" />
<meta http-equiv="expires" content="0" />
