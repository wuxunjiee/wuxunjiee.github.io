# 深入响应式原理

# 前言

为什么修改了数据，就影响了试图的变化？

修改试图的方式：

  • 直接操作 DOM

  • 例如 JQuery

    • 还是在操作 DOM
    • 只不过是提高了操作 DOM 的效率

  • 数据驱动试图方式（MVVM）:通过数据操作来影响视图更新

    • 最早是微软的一个技术：WPF,一个专门用来开发 Winodws 图形软件的技术
    • 前端框架 Angular 第一个把 MVVM 带入前端

# 使用代码实现

如何实现修改一个对象成员就修改了 DOM

JavaScript数据劫持或者说 JavaScript 对象属性拦截器

说白了就是我们能监视到 JavaScript 对象数据成员的访问和修改

什么是数据劫持？

说白了就是：观察数据的变化，执行特定的业务逻辑

  • Object.defineProperty

    • es5 中的 api
    • Vue3 之前都是使用的这个 api

  • Proxy

    • es6 新增的 api
    • Vue3 使用的这样 api
    • Proxy 比 Object.defineProperty 功能和性能更好

Object.defineProperty 实现

const user = {};
// 如何给对象添加数据成员？
// user.xxx = xxx 或者Object.defineProperty
// 参数1: 源数据 user 参数2: 属性名 name 参数3: 配置对象 function
// 注意点： 该属性不存储数据，它只是代理了name的访问和修改，仅此而已。如果要存储数据，需要借助一个普通变量 （_name）
let _name = "";
Object.defineProperty(user, "name", {
  // 当你访问 user.name时候会自动调用 get 方法
  get() {
    return _name;
  },
  // 当你对 user.name 赋值时候会自动调用 set 方法
  set(val) {
    document.querySelector("div").innerHTML = val;
    _name = val;
  },
});

这种方式每次都需要额外定义一个变量，接下来我们来对它进行封装

假设封装的方法名为 defineReactive,我们希望这样调用 defineReactive(对象，属性，值)

代码实现

const user = {};
defineReactive(user, "age", 18);
function defineReactive(obj, key, value) {
  Object.defineProperty(obj, key, {
    get() {
      return value;
    },
    set(newVal) {
      if (newVal !== value) value = newVal;
    },
  });
}

此时我们再去监测一个对象的属性变化让它存储数据不需要再额外定义一个变量接收了，可是我们会考虑这样一个问题

如果对象中的成员也是对象了，那么这种情况下我们怎么监测到对象中对象属性的变化？

代码实现（使用递归）

// 数据源
const data = {
  name: "张三",
  user: {
    age: 18,
    sex: "male",
  },
};
// 对 data 中所有数据成员进行数据劫持
observe(data);
function observe(data) {
  if (!data || typeof data !== "object") return;
  //  取出所有属性进行遍历，对属性成员进行代理（拦截、观察）操作
  Object.keys(data).forEach((key) => {
    defineReactive(data, key, data[key]);
  });
}
function defineReactive(obj, key, value) {
  observe(value);
  Object.defineProperty(obj, key, {
    get() {
      return value;
    },
    set(newVal) {
      console.log(`${key}属性变化了`);
      if (newVal !== value) value = newVal;
    },
  });
}

Proxy 实现

Proxy 可以理解成，在目标对象之前架设一层"拦截"，外界对该对象的访问，都必须先通过这层拦截，因此提供了一种机制，可以对外界的访问进行过滤和改写。Proxy 这个词的原意就是代理，用在这里表示它来"代理"某些操作，可以译为"代理器"

语法

let p = new Proxy(target, handler);

Proxy 对象的所有用法都是上面这种形式，不同的是handler参数的写法。

  • new Proxy()表示生成一个 Proxy 实例

  • target 参数表示所有要拦截的目标对象

    • 可以是任意类型的对象，包括数据，函数，甚至可以是另一个 Proxy

  • handler 参数也是一个对象，用来定制拦截行为

    • 其属性是当执行一个操作时定义代理的行为的函数

代码实现

const data = {
  name: "张三",
  user: {
    age: 18,
    sex: "male",
  },
};
//  参数1: 数据源 参数2: 配置对象 返回值： 代理结果数据
let p = new Proxy(data, {
  //  target: 被代理的目标对象
  //  property: 访问的属性名
  get(target, property) {
    console.log(`${property}属性被访问了`);
    return target[property];
  },
  set(target, property, value) {
    console.log(`${property}属性被修改了`);
    target[property] = value;
  },
});

使用 Proxy 修改了数据数据会存储起来，也就是说 p.name="李四",data.name 值也会修改为"李四",如果使用 Object.defineProperty 则需要定义个变量，或者对它进行封装才能实现

使用 Proxy 代理所有对象成员（监测到对象中属性也是对象的变化）

const data = {
  name: "张三",
  user: {
    age: 18,
    sex: "male",
  },
};
let propsData = observe(data);
function observe(data) {
  if (!data || typeof data !== "object") return;
  let p = new Proxy(data, {
    //  target: 被代理的目标对象
    //  property: 访问的属性名
    get(target, property) {
      console.log(`${property}属性被访问了`);
      return target[property];
    },
    set(target, property, value) {
      console.log(`${property}属性被修改了`);
      target[property] = value;
    },
  });
  for (const key in data) {
    const item = data[key];
    if (data.hasOwnProperty(key) && typeof item === "object") {
      data[key] = observe(item);
    }
  }
  return p;
}
