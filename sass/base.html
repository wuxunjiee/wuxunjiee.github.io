# 基础知识

# 变量（Variables）

$font-stack: Helvetica, sans-serif;
$primary-color: #333;

body {
  font: 100% $font-stack;
  color: $primary-color;
}

使用场景

例如在公司项目中，你的项目中主题色由#333 改为了#999，你不可能一个一个把所有的主题颜色全部替换为#999,那样是不是显得太繁琐了，你只需通过$定义一个变量，并把主题色赋值给这个变量，当主题色更改后只需要给变量重新赋值就可以实现所有的主题颜色更改了。

# 引用父选择符（&）

a {
  font-weight: bold;
  text-decoration: none;
  &:hover {
    text-decoration: underline;
  }
  body.firefox & {
    font-weight: normal;
  }
}

被编译为

a {
  font-weight: bold;
  text-decoration: none;
}
a:hover {
  text-decoration: underline;
}
body.firefox a {
  font-weight: normal;
}

# 导入（@import）

需要被导入的文件，文件名：_base.scss

html,body {
    padding: 0
    margin: 0
}

导入的文件

@import "base";

a {
  text-decoration: none;
}

被编译后的文件

html,
body {
  padding: 0;
  margin: 0;
}

a {
  text-decoration: none;
}

Sass 扩展了 CSS 的 @import 规则，让它能够引入 SCSS 和 Sass 文件。 所有引入的 SCSS 和 Sass 文件都会被合并并输出一个单一的 CSS 文件。 另外，被导入的文件中所定义的变量或 mixins 都可以在主文件中使用。

注意点： 文件名下划线（_）,杠（-）命名，@import 可以省略这些符号，以及.sass,.scss 后缀可以省略，见上述代码段 base 文件

好处： 缓存 ，默认情况下，Sass 会对编译过的模板（template）和 partials 进行缓存。 这将明显加快大量 Sass 文件的重新编译速度， 并且在 Sass 模板被切割为多个文件并通过 @import 引入形成一个大文件时效果最好。

# 混入（mixins）

Mixin 是 Sass 中用来方便地复用代码的方法，你可以简单点理解成 函数 ，可以 传递参数 ，参数名以$符号开始，多个参数以逗号分开，也可以给参数设置默认值，返回的是一组 CSS 属性或代码。

使用场景一： 上述为浏览器私有前缀的兼容性写法，$radius 为参数

@mixin border-radius($radius) {
  -webkit-border-radius: $radius;
  -moz-border-radius: $radius;
  -ms-border-radius: $radius;
  border-radius: $radius;
}

.box {
  @include border-radius(10px);
}

被编译为

.box {
  -webkit-border-radius: 10px;
  -moz-border-radius: 10px;
  -ms-border-radius: 10px;
  border-radius: 10px;
}

使用场景二：

@mixin triangle($direction, $size, $borderColor) {
  content: "";
  height: 0;
  width: 0;
  @if ($direction == top) {
    border-bottom: $size solid $borderColor;
    border-left: $size dashed transparent;
    border-right: $size dashed transparent;
  } @else if ($direction == right) {
    border-left: $size solid $borderColor;
    border-top: $size dashed transparent;
    border-bottom: $size dashed transparent;
  } @else if ($direction == bottom) {
    border-top: $size solid $borderColor;
    border-left: $size dashed transparent;
    border-right: $size dashed transparent;
  } @else if ($direction == left) {
    border-right: $size solid $borderColor;
    border-top: $size dashed transparent;
    border-bottom: $size dashed transparent;
  }
}

.box {
  @include triangle(top, 2px, #333);
}

被编译为

.box {
  content: "";
  height: 0;
  width: 0;
  border-bottom: 2px solid #333;
  border-left: 2px dashed transparent;
  border-right: 2px dashed transparent;
}

# 继承（@extend）

%message-shared {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  @extend %message-shared;
  background-color: green;
}

.warning {
  @extend %message-shared;
  background-color: yellow;
}

.error {
  @extend %message-shared;
  background-color: red;
}

被编译为

.error,
.warning,
.success {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  background-color: green;
}

.warning {
  background-color: yellow;
}

.error {
  background-color: red;
}

使用场景： 比如常用的 Tooltip 提示消息,其实它的宽，高，显示的位置，边框都是通用的，如果是错误提示消息显示为红色，成功消息显示为绿色，只是更改了背景颜色，所以可以使用继承对公共样式进行复用。上面有个细节是我使用了Sass3.2中引入了一个新的特性 选择占位符% placeholder。

我会在下一节讲解为什么要使用它，使用它的好处在哪？

# 占位符选择器（%）

一些废话（此处可以跳过）： 在刷 sass 官方文档的时候其实我对%选择器没太看明白，或者说没有理解太透彻，于是去找了下大漠老师讲解 css 的博客去查阅了一下，我把一些见解和坑分享在下面。

不使用%选择器的代码演示

.icon {
  transition: background-color ease 0.2s;
  margin: 0 0.5em;
}
.error-icon {
  @extend .icon; /*错误图标指定的样式... */
}
.info-icon {
  @extend .icon; /* 信息图标指定的样式... */
}

被编译为

.icon,
.error-icon,
.info-icon {
  transition: background-color ease 0.2s;
  margin: 0 0.5em;
}
.error-icon {
  /*错误图标指定的样式... */
}
.info-icon {
  /* 信息图标指定的样式... */
}

如果我们在 HTML 中从来不使用 .icon ，其存在的唯一目的就是为了扩展吗？这似乎比我们需要的基本样式稍大些，因为我们将永远不会使用.icon 对应的样式。其实在 Sass3.2 开始，我们可以通过使用选择器占位符%placeholder 来解决这种现象。

使用%选择器的代码演示

%icon {
  transition: background-color ease 0.2s;
  margin: 0 0.5em;
}
.error-icon {
  @extend %icon; /*错误图标指定的样式... */
}
.info-icon {
  @extend %icon; /* 信息图标指定的样式... */
}

被编译为

.error-icon,
.info-icon {
  transition: background-color ease 0.2s;
  margin: 0 0.5em;
}
.error-icon {
  /*错误图标指定的样式... */
}
.info-icon {
  /* 信息图标指定的样式... */
}

注意 ，编译出来的 CSS 代码中将不再包括.icon 了。

坑： 使用@extend 调用定义好的选择器占位符%placeholder 有所限制，他不能在不同的@media 中运行。否则会报错。因为@extend 是将一个选择器样式扩展到另一个选择器当中，而实际上在不同的@media 中却无需复制这些样式。

# 运算

运算常用于做移动端浏览器适配，根据不同的屏幕宽度显示不同的样式。使用很简单，没有什么好特别说明的，下面列一个坑出来

坑： 错误写法

.self-adaption {
  width: 100px / 2;
}

被编译为

.self-adaption {
  width: 100px/2;
}

我测了一下只有除法会这样，至于为什么，我也不知道，反正我也是醉了，如果是除法需要包一个 括号 ，如下。

.self-adaption {
  width: (100px / 2);
}

被编译为

.self-adaption {
  width: 50px;
}
