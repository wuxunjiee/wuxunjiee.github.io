# SEO 优化

# 什么是 SEO

SEO(Search Engine Optimization):搜索引擎优化。是一种方式：利用搜索引擎的规则提高网站在有关搜索引擎内的自然排名。目的是让其在行业内占据领先地位，获得品牌收益。很大程度上是网站经营者的一种商业行为，将自己或自己公司的排名前移。

我们使用 Vue 开发出来的应用是 SPA 单页面应用程序 它最大的缺点：不利于 SEO

如何解决？ 服务端渲染，在服务端完成页面的解析替换，发送给浏览器的直接就是结果。

# SSR(服务端渲染)：服务器直接生成 HTML 文档返回给浏览器

  • 好处： 响应速度快，有利于 SEO
  • 缺点： 前后端代码混在一起，难以开发和维护，不适合进行前后端分离开发

总结

  • 服务端返回一个空的页面结果

  • 客户端加载执行对应的 JavaScript 脚本

  • 客户端操作 DOM 完成解析替换

代码演示

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>CSR</title>
  </head>
  <body>
    <div id="app">
      <h1>{{message}}</h1>
    </div>
    <script src="https://cdn.jsdelivr.net/npm/vue/dist/vue.js"></script>
    <script>
      new Vue({
        el: "#app",
        data: {
          message: "hello world",
        },
      });
    </script>
  </body>
</html>

# CSR(客户端渲染)：页面出事加载的 HTML 文档中无内容，需要下载执行 JS 文件，由浏览器动态生成页面，并通过 JS 进行页面交互事件与状态管理

  • 好处： 页面导航不用刷新整个页面，用户体验好，有利于前后端分离开发
  • 缺点： 不利于 SEO(因为单页应用都是使用客户端渲染的方式)，还有首次响应慢（第一次要加载大量的公共资源）

因为传统的搜索引擎只会从 HTML 中抓取数据，这会导致前端渲染的页面无法被渲染。一般页面首屏时间大概由80%消耗在网络上，剩下的时间在后端抓取数据以及浏览器渲染，显然要优化后面的 20%是比较困难的，优化网络时间是效果最明显的手段。传统的 AJAX 请求是先请求 js，再由 js 发送数据请求，两项时间加上浏览器渲染时间才是首屏时间。而 SSR 能将两个请求合并为一个。

当然缺点就是增加了服务器端负载，服务端和浏览器开发环境的差异带来的问题，例如 document 对象找不到。

代码演示

安装依赖：

// 创建http服务
npm i express

// 服务端模板引擎
npm i art-template express-art-template

服务端代码：

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>ssr</title>
  </head>
  <body>
    <!-- art-template模板语法 {{}}用户数据绑定 数据从哪儿来？ res.render方法的第二个参数 -->
    <h1>{{message}}</h1>
  </body>
</html>

const express = require("express");

const app = express();

// 设置使用 art-template 模版引擎
// 服务端也可以使用模板引擎，因为模板引擎的本质就是在进行字符串替换
app.engine("html", require("express-art-template"));

// 当请求 / 的时候，渲染 index.html 页面给客户端
app.get("/", (req, res) => {
  // 渲染views/index.html 给客户端
  // 提示：它默认就是从views目录中查找文件，这是一个固定规则
  // render 方法的作用：
  // 1.读取index.html文件（字符串）
  // 2.模板引擎解析替换 它会把整个HTML文件内容都当作模板字符串 数据就是参数2 返回解析替换的结果
  // 3.把结果发送给客户端浏览器
  res.render("index.html", {
    message: "hello world",
  });
});

app.listen(3000, () => console.log("http://localhost:3000"));

查看

可以看到客户端渲染多去加载一个 js 文件去解析替换字符串，而服务端渲染直接把解析后的结果返回回来给客户端展示，比客户端渲染要少跑一次，所以首屏渲染会快一些

# 如何选择

一般注重 SEO 的新闻站点或者企业官网这类非强交互的页面建议使用服务端渲染，而像后台管理系统建议使用客户端渲染

# Nuxt

基于 Vue 的服务端渲染框架

关于 Nuxt 中页面的渲染

在浏览器中打开的第一个页面的服务端渲染过来的

  • 例如我直接访问的首页（浏览器直接输入地址访问），则首页就是我们访问的第一个页面
  • 例如我直接访问的登录页（浏览器直接输入地址访问），则登录页就是我们访问的第一个页面

之后在页面上通过点击路由切换都是在客户端进行的，也就是说是客户端渲染，这是 Nuxt 的核心特点

asyncData

asyncData 是 Nuxt 中扩展的一个生命周期钩子，它会在整个组件渲染前最先调用

我们要注意的是就是把需要 SEO 优化的数据请求写到这里

注意：该方法因为是在服务端执行的，所以没有实例 this

最终 Nuxt 会将 asyncData 和 data 中的数据组合在一个对象中供我们使用（例如 asyncData 中 return {name: '张三'},data: {age: 18}）最终 data 中的数据为{name: '张三',age:18}

总结：如果页面中通过接口渲染的数据需要 SEO,则把请求写到 asyncData 函数中，否则可以写在 created 函数中。目前只有beforeCreate 钩子是既在服务端执行又在客户端执行，我们可以把急需要在客户端也需要在服务端执行的代码写在这个钩子里

最后需要再次强调，Nuxt 的渲染核心特点是第一次进入页面走服务端渲染，后面的页面切换都是走客户端渲染，我们可以在控制台验证，第一次进入页面在 network 无法看到接口请求的，从点击路由进来却可以在 network 看到这个接口请求了。并且 asyncData 方法会在两个时机触发，一个是服务端，另一个是路由切换（在客户端路由切换执行了asyncData所以渲染的数据也是支持了SEO）。虽然后续的页面切换是客户端渲染但是不影响 SEO 优化，因为爬虫来爬取网页没有路由切换一说，都是相当于第一次访问页面来爬取网页内容。这也是 Nuxt 的做的一种优化，既做了 SEO 优化，又保留了单页应用的体验。
