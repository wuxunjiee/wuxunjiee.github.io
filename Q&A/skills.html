# axios 归纳

# axios.create 使用场景

  • axios.create 相当于复制了一个 axios，复制的 axios 和原本的 axios 是相互独立互不影响的，设置自己独立的 baseURL,响应拦截器，请求拦截器等等

  • 使用场景：在开发中，登录时我们根据 token 去记录登录态，token 出于安全考虑是有时效性的，一般可能是 2 小时左右，可是每 2 小时就让用户重新登录，用户体验不好。会有一个 refresh_token，时效性会长很多，一般可能会 12 小时左右，具体后端定义。

此时因为请求 token 和 refresh_token 的请求头不一致，所以我们创建两个独立的 axios 来处理

headers: {
  Authorization: `Bearer ${user.token}`;
}
headers: {
  Authorization: `Bearer ${user.refresh_token}`;
}

# keep-alive

# keep-alive 是什么？

  • keep-alive 会缓存组件
  • 组件缓存之后，生命周期不再执行了
  • 如果不需要某些组件缓存，可以指定 include 或者 exclude 来包含或者排除 Î

组件间切换会先销毁原来的组件，再加载新的组件。如果使用了 keep-alive 缓存该组件，则不会销毁。

<keep-alive include="需要缓存的组件name">
      <router-view />
</keep-alive>

# keep-alive 妙用

开发中有一个很常见的场景，假设我们现在有两个页面：文章列表页面，文章详情页面。我们一般会在进入文章列表页面希望调接口加载最新的数据，但在进入详情页面的时候我们希望在访问文章详情后换回能记录住我们之前的 tabbar 栏信息，和滚动条滑动的位置信息，而不是又从头开始加载页面。

那么问题来了，文章列表这一个组件我们并不希望它一直是缓存着的，所以我们就要用到条件缓存，在特定的条件下才被缓存

好的话不多说，直接上代码

<keep-alive :include="$store.state.cachedPages">
      <router-view />
</keep-alive>

store 容器中代码如下

removeCachePage (state, pageName) {
  const index = state.cachedPages.indexOf(pageName)
  if (index !== -1) {
    state.cachedPages.splice(index, 1)
  }
},

addCachePage (state, pageName) {
  if(!state.cachedPages.includes(pageName)){
    state.cachedPages.push(pageName)
  }
}

文件列表组件代码如下

beforeRouteLeave (to, from, next) {
  // 导航离开该组件的对应路由时调用
  // 可以访问组件实例 `this`
  // 如果是去 文章详情 页面，则缓存
  const { name: pageName } = this.$options
  if (to.name === 'article') {
    this.$store.commit('addCachePage', pageName)
  } else {
    // 如果是去其他页面，则不缓存
    this.$store.commit('removeCachePage', pageName)
  }
  next()
}

# 测试打包结果

当我们把vue项目 yarn build（yarn可以省略run）后需要将dist放到一个Web服务器进行测试

  • Nginx

  • Apache

  • tomcat

  • ...

  • Node.js

不过推荐使用Vue官网推荐的一个命令行http服务工具：serve

安装：

yarn global add serve

查看是否安装成功：

serve --version

使用：

serve -s dist

serve默认占用5000端口并启动一个服务
