# placeholder 问题

问题描述：表单元素的 placeholder 属性在 IE 上不支持

解决方案：yarn add ie-placeholder

npm 地址：https://www.npmjs.com/package/ie-placeholder

# 文字省略号问题

问题描述 ：在开发时常常会遇到单行或多行文本溢出显示省略号问题，以往的项目中我们通常使用 css 代码去实现，代码如下

text-overflow: ellipsis;
overflow: hidden;
display: -webkit-box;
-webkit-line-clamp: $line;
// 超出2行显示省略号的样式有个 -webkit-box-orient: vertical 的属性
// 但打包出来之后被移除了
// 添加autoprefixer: off可以保持不被移除
/*! autoprefixer: off */
-webkit-box-orient: vertical;

这段代码主要使用 line-clamp 这个 css3 属性来控制文本行数，不过遗憾的是该属性在 IE 中对也不支持。

解决方案

方案 1：这个方案是我自己想出来的，封装一个自定义指令，通过 js 去计算解决这个问题。项目前期使用这个指令并未发现有任何问题，但在一些特殊的交互效果下会有获取不到宽度的 Bug，因此该方案不推荐在项目中使用，不过还是简单描述下方案思路，供大家参考。

export default {
  /**
   * @anthor 吴汛杰
   * @description 解决低版本浏览器文字省略号失效问题
   */
  update(el, binding) {
    const FONT_SIZE =
      window.getComputedStyle(el).fontSize.replace("px", "") - 0;
    const { offsetWidth: WIDTH, innerText: CONTENT } = el;
    if (!WIDTH) return;
    for (let i = 0; i < CONTENT.length; i++) {
      if (FONT_SIZE * i >= WIDTH * (binding.value || 1)) {
        el.innerText = CONTENT.substr(0, i - 1) + "...";
        return;
      }
    }
  },
};

使用：html 标签上使用 v-指令名称="需要几行展示的数字" 示例：

<!-- 2行展示，超出显示省略号 -->
<!-- 如果是一行超出显示省略号可以简写为 v-ellipsis 因为代码内部给了默认值为1 -->
<p v-ellipsis="2"></p>

思路简述：通过拿到字体的 font-size，和文本内容，和布局盒子的宽度。循环时当文档的长度超出了文本内容的最大宽度就截取该坐标位置并拼接'...'字符串。其实这段代码还可以在里面判断浏览器版本，对于不支持通过 css 实现的才通过指令 js 实现，这样处理，性能更好。

方案 2：使用别人开源的插件 vue-clamy 推荐

github 地址：https://github.com/clampy-js/vue-clampy

使用：

yarn add @clampy-js/vue-clampy
<script>
import clampy from '@clampy-js/vue-clampy';
import Vue from 'vue';

Vue.use(clampy);

export default {
  name: 'app',
  directives: {
    clampy
  }
};
</script>
<template>
  <p v-clampy="3">Long text to clamp here</p>
</template>

使用情况：在潍柴项目 PC 端大量使用暂未发现有任何 Bug 。推荐使用。

# Vue 项目某些特殊插件源码未进行 babel 转译问题

问题描述： 例如'vue-number-animation','js-base64'这类插件没有对里面的 js 代码进行 babel 转译为 es5，导致在 IE 或其他不支持 es6 语法的浏览器报错。造成整个页面显示空白。

解决方案：

// vue.config.js文件
// 数组中加上你需要babel转译的插件名即可
transpileDependencies: ["vue-number-animation", "js-base64"];

# el-drawer 组件在部分浏览器出现首次打开抖动问题

问题描述：在 element 2.13 版本后 el-drawer 组件有个官方产生的 Bug 未解决。经测试后在 IE,Safari,QQ 浏览器上均会出现首次点开抖动问题

解决方案：

<!-- 在组件html代码最上面写一个看不见的input元素即可解决此问题 -->
<input style="opacity: 0;position: absolute;left: -9999px" />

参考文档

https://github.com/ElemeFE/element/issues/18270

https://www.freesion.com/article/3495502785/ （内有问题复现 gif）

# el-input 组件在 IE9 上事件不触发问题

问题描述：el-input 组件经测试发现在 IE9 上使用退格/删除键不会触发 input 事件

解决方案：可通过触发 change 事件去解决这个问题

参考文档：https://github.com/ElemeFE/element/issues/4180

# el-upload 组件 IE9 上不支持

问题描述：饿了么的文件上传组件内部是使用的 FormData 去实现，但是 FormData 却只支持到 IE10，因为在 IE9 上点击文件上传会无反应

解决方案：

使用 vue-upload-component 插件去解决 github 地址：https://github.com/lian-yue/vue-upload-component 在潍柴 PC 端有对这个插件根据业务进行二次封装，因为封装的组件和业务有牵连，耦合性较低，所以不贴出代码。后续有时间可对该组件封装成通用性组件再补充代码。

# v-infinite-scroll 配合 el-drawer 组件使用失效问题

问题描述：饿了么组件库官方提供了 infinite-scroll 指令去实现无限滚动效果，但是在开发中该指令配合 el-drawer 组件使用会失效，于是在项目中封装了自定义指令去解决。

解决方案：

export default {
  /**
   * @anthor 吴汛杰
   * @description 解决v-infinite-scroll配合el-drawer组件使用会失效问题
   */
  bind(el, binding) {
    let SCROLL_DOM = el;
    let scrollPosition = 0;
    SCROLL_DOM.addEventListener("scroll", function () {
      // 当前的滚动位置 减去  上一次的滚动位置
      // 如果为true则代表向上滚动，false代表向下滚动
      let flagToDirection = this.scrollTop - scrollPosition > 0;
      // 记录当前的滚动位置
      scrollPosition = this.scrollTop;
      const LIMIT_BOTTOM = 100;
      // 记录滚动位置距离底部的位置
      let scrollBottom =
        this.scrollHeight - (this.scrollTop + this.clientHeight) < LIMIT_BOTTOM;
      // 如果已达到指定位置则触发
      if (scrollBottom) {
        // 将滚动行为告诉组件
        binding.value(flagToDirection);
      }
      console.log(flagToDirection ? "滚动方向：下" : "滚动方向：上");
    });
  },
};

# 获取浏览器名称方法代码

export function getBrowserVersion() {
  const userAgent = navigator.userAgent; //取得浏览器的userAgent字符串
  const isOpera = userAgent.indexOf("Opera") > -1; //判断是否Opera浏览器
  const isIE =
    userAgent.indexOf("compatible") > -1 &&
    userAgent.indexOf("MSIE") > -1 &&
    !isOpera; //判断是否IE浏览器
  const isEdge = userAgent.indexOf("Edge") > -1; //判断是否IE的Edge浏览器
  const isFF = userAgent.indexOf("Firefox") > -1; //判断是否Firefox浏览器
  const isSafari =
    userAgent.indexOf("Safari") > -1 && userAgent.indexOf("Chrome") == -1; //判断是否Safari浏览器
  const isChrome =
    userAgent.indexOf("Chrome") > -1 && userAgent.indexOf("Safari") > -1; //判断Chrome浏览器

  if (isIE) {
    const reIE = new RegExp("MSIE (\\d+\\.\\d+);");
    reIE.test(userAgent);
    const fIEVersion = parseFloat(RegExp["$1"]);
    if (fIEVersion == 7) {
      return "IE7";
    } else if (fIEVersion == 8) {
      return "IE8";
    } else if (fIEVersion == 9) {
      return "IE9";
    } else if (fIEVersion == 10) {
      return "IE10";
    } else if (fIEVersion == 11) {
      return "IE11";
    } else {
      return "0";
    } //IE版本过低
    // return 'IE'
  }
  if (isOpera) {
    return "Opera";
  }
  if (isEdge) {
    return "Edge";
  }
  if (isFF) {
    return "FF";
  }
  if (isSafari) {
    return "Safari";
  }
  if (isChrome) {
    return "Chrome";
  }
}

# Safari 浏览器获取元素宽度精度比其他浏览器要高

经测试发现，同一个盒子 Chrome 及其他浏览器元素宽度为 20px，Safari 宽度可能为 20.532px，最终 Safari 显示的为 21px。 这样在有些需求中，前端通过动态计算去显示页面元素的时候 Safari 和其他浏览器的显示效果是有细微差异的，这是正常情况。
