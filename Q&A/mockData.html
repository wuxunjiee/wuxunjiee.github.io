# 前端 Mock 数据方案及开发中小技巧分享

我们在开发中可能会常常遇到前端页面开发完成，后端接口还未能提供出来进行接口联调的情况。这时候我们可能会根据后端定义好的请求路径和返回字段先造一点假数据开发。

# 那么我们有哪些造假数据的方法呢？

  • 本地写 JSON 文件，在这个 JSON 文件中定义你需要的数据，然后请求这个 JSON 文件
// api/shop.js 文件
const __products = [
  { id: 1, title: "iPad 4 Mini", price: 500.01, inventory: 2 },
  { id: 2, title: "iPhone XsMAx", price: 3000.99, inventory: 10 },
  { id: 3, title: "HuaWei MatePro", price: 2699.01, inventory: 5 },
];

export const getAllProducts = () => {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve(__products);
    }, 100);
  });
};

import * as shop from "@/api/shop";
export default {
  methods: {
    async initData() {
      await shop.getAllProducts();
    },
  },
};

例如我需要一个商品列表的数据我可以选择上面这种方案。

优点：简单，快捷。适用于临时需要使用数据解决方案

缺点：数据不够灵活，如果我需要 100 条数据，需要手动复制 100 条

# 使用 Node 写一个返回数据的服务

// server/index.js
const express = require("express");

const app = express();

// 解决跨域问题
app.use((req, res, next) => {
  res.header("Access-Control-Allow-Origin", "*");
  if (req.method === "OPTIONS") return res.send();
  next();
});

app.get("/getAllProducts", (req, res) => {
  res.json({
    code: 0,
    data: {
      products: [
        { id: 1, title: "iPad 4 Mini", price: 500.01, inventory: 2 },
        { id: 2, title: "iPhone XsMAx", price: 3000.99, inventory: 10 },
        { id: 3, title: "HuaWei MatePro", price: 2699.01, inventory: 5 },
      ],
    },
  });
});

app.listen("3000", () => {
  console.log("http://localhost:3000");
});

//api/index.js
import axios from "axios";

axios.defaults.baseURL = "http://localhost:3000";

export const getAllProducts = () => axios.get("/getAllProducts");

import { getAllProducts } from "@/api";
export default {
  methods: {
    async initData() {
      await shop.getAllProducts();
    },
  },
};

优点：简单，快捷。相比于上一个解决方案我只需要将请求路径和返回字段定义成和后端一致，等后端接口好了前端只需简单测试一下，不用改动什么代码

缺点：数据不够灵活，如果我需要 100 条数据，需要手动复制 100 条

当然，其实还有很多其他解决方案，例如使用 json-server 也可以 mock 数据，这里不再一一介绍。下面主要介绍一款使用人数较多，评价不错的方案Mock.js

# Mock.js 是什么？

生成随机数据，拦截 Ajax 请求(摘自官网描述)

官网的描述已经非常清晰了。下面我们简单使用一些 Mock.js 来对它的这两项特性有个更充分的认识和理解

优势

  • 采用 json 数据模拟，生成数据比较繁琐，也比较有局限性，没办法达到增删改查
  • 采用 mockJs 进行模拟数据，可以模拟各种场景（get、post）生成接口，并且随机生成所需数据，还可以对数据进行增删改查

  • 在项目中新建 mock 文件

    //引入mock模块
import Mock from "mockjs";

  • 将 mock 文件在 main.js 中导入

    import Vue from "vue";
import App from "./App.vue";
if (process.env.NODE_ENV !== "production") require("./mock");

Vue.config.productionTip = false;

new Vue({
  render: (h) => h(App),
}).$mount("#app");
# mock 语法
# 生成字符串

  • 生成指定次数字符串

    import Mock from "mockJs";
const data = Mock.mock({
  "string|4": "哎呦！",
});

  • 生成指定范围长度字符串

    const data = Mock.mock({
  "string|1-8": "哎呦！",
});
# 生成文本

  • 生成一个随机字符串

    const data = Mock.mock({
  string: "@cword",
});

  • 生成指定长度和范围

    const data = Mock.mock({
  string: "@cword(1)",
  str: "@cword(10,15)",
});
# 生成标题和句子

  • 生成标题和句子

    const data = Mock.mock({
  title: "@ctitle",
  sentence: "@csentence",
});

  • 生成指定长度的标题和句子

    const data = Mock.mock({
  title: "@ctitle(8)",
  sentence: "@csentence(50)",
});

  • 生成指定范围的

    const data = Mock.mock({
  title: "@ctitle(5,8)",
  sentence: "@csentence(50,100)",
});
# 生成段落

  • 随机生成段落

    const data = Mock.mock({
  content: "@cparagraph()",
});
# 生成数字

  • 生成指定数字

    const data = Mock.mock({
  "number|80": 1,
});

  • 生成范围数字

    const data = Mock.mock({
  "number|1-999": 1,
});
# 生成增量 id

  • 随机生成标识

    const data = Mock.mock({
  id: "@increment()",
});
# 生成姓名-地址-身份证号

  • 随机生成姓名-地址-身份证号

    const data = Mock.mock({
    name: '@cname()',
    idCard: '@id()',
    address: '@city(true)'
})​

    # 开发技巧分享

    • vscode 插件推荐

    GitLens

    有时我们需要知道某一个功能某一块业务是谁负责编写的，我们只需要把鼠标放在当前这行代码上就会知道作者，时间，commit-message信息。非常方便

    • ngrok 免费的内外穿透工具

    使用ngrok

  • 注册
  • 下载
  • 解压 解压出来会得到一个可执行程序文件，建议把该程序文件访问用户目录中
  • 配置用户凭证 ./ngrok authtoken 用户凭证
  • 查看使用帮助 ./ngrok help
  • 开启代理指定本地web服务 ngrok http 80 （本地需要有此端口服务才行）

比如我在开发潍柴项目时，因为项目需要兼容ie9浏览器，但是我的电脑上没有这个环境。我可能需要把项目挂在外网上，让别人在ie9访问一下，帮我看下项目有没有问题。这个时候我们不需要花钱花时间花精力去维护一个服务器，只需简单的跑一行命令，别人就可以访问了。当然它也有缺点：就是访问速度非常慢
